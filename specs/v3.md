# V3 Specification — Resume Management + jdUrl Field

## Overview

Two features:

1. **Resume management** — Upload, store, and retrieve resumes (master + job-tailored). Replace the hardcoded `RESUME_SUMMARY` in `ResumeScorerAgent` with real resume data from the database.
2. **jdUrl field** — Add a reference URL field to jobs so users can link back to the original job posting. No scraping — purely a stored reference.

---

## Feature 1: Resume Management

### Data Model

#### Migration: `V6__create_resumes.sql`

```sql
CREATE TABLE resumes (
    id            UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    job_id        UUID         REFERENCES jobs(id) ON DELETE CASCADE,
    file_name     VARCHAR(255) NOT NULL,
    file_content  BYTEA        NOT NULL,
    content_text  TEXT         NOT NULL,
    created_at    TIMESTAMP    NOT NULL DEFAULT now()
);

CREATE INDEX idx_resumes_job_id ON resumes(job_id);
```

- `job_id` is **nullable**: `NULL` = master resume, non-null = tailored resume for that job.
- `file_content` stores the original uploaded file as binary (PDF, DOCX, or plain text).
- `content_text` stores the extracted plain text used by the scoring pipeline.
- The **current master resume** is the most recent row where `job_id IS NULL`, ordered by `created_at DESC`.
- Multiple master resumes can exist (history). Latest wins.
- Each job can have multiple tailored resumes. Latest wins.

#### JPA Entity: `Resume.java`

Create in `src/main/java/com/jobtracker/model/Resume.java`.

Follow the exact patterns from `Score.java` and `AgentRun.java`:

```java
package com.jobtracker.model;

import jakarta.persistence.*;
import java.time.LocalDateTime;
import java.util.UUID;

@Entity
@Table(name = "resumes")
public class Resume {

    @Id
    @GeneratedValue(strategy = GenerationType.UUID)
    private UUID id;

    @Column(name = "job_id")
    private UUID jobId;  // nullable — null means master resume

    @Column(name = "file_name", nullable = false)
    private String fileName;

    @Column(name = "file_content", nullable = false)
    @Basic(fetch = FetchType.LAZY)
    private byte[] fileContent;

    @Column(name = "content_text", nullable = false, columnDefinition = "TEXT")
    private String contentText;

    @Column(name = "created_at", nullable = false, updatable = false)
    private LocalDateTime createdAt;

    @PrePersist
    void prePersist() {
        this.createdAt = LocalDateTime.now();
    }

    // Getters and setters for all fields (no Lombok).
    // id has getter only (no setter — auto-generated).
    // Follow the same pattern as Score.java.
}
```

**Important notes:**
- `fileContent` uses `@Basic(fetch = FetchType.LAZY)` to avoid loading the binary blob on every query.
- `jobId` is a plain UUID, NOT a `@ManyToOne` JPA relationship. Same pattern as `Score.jobId` and `AgentRun.jobId`.
- No soft delete on resumes.

#### Repository: `ResumeRepository.java`

Create in `src/main/java/com/jobtracker/repository/ResumeRepository.java`.

```java
package com.jobtracker.repository;

import com.jobtracker.model.Resume;
import org.springframework.data.jpa.repository.JpaRepository;
import java.util.Optional;
import java.util.UUID;

public interface ResumeRepository extends JpaRepository<Resume, UUID> {

    // Latest master resume (job_id IS NULL, most recent)
    Optional<Resume> findFirstByJobIdIsNullOrderByCreatedAtDesc();

    // Latest tailored resume for a specific job
    Optional<Resume> findFirstByJobIdOrderByCreatedAtDesc(UUID jobId);
}
```

Follow the query method naming pattern from `ScoreRepository.findFirstByJobIdOrderByCreatedAtDesc`.

#### Endpoints:

| Method | Path | Description | Request | Response | Status |
|--------|------|-------------|---------|----------|--------|
| `POST` | `/api/v1/resume` | Upload master resume | Multipart: `file` (required) | `ResumeResponse` | `201` |
| `GET` | `/api/v1/resume` | Get current master resume metadata | — | `ResumeResponse` (nullable) | `200` |
| `GET` | `/api/v1/resume/download` | Download current master resume file | — | Binary file with Content-Disposition | `200`, `404` |
| `POST` | `/api/v1/jobs/{id}/resume` | Upload tailored resume for a job | Multipart: `file` (required) | `ResumeResponse` | `201`, `404` |
| `GET` | `/api/v1/jobs/{id}/resume` | Get tailored resume metadata for a job | — | `ResumeResponse` (nullable) | `200`, `404` |
| `GET` | `/api/v1/jobs/{id}/resume/download` | Download tailored resume file | — | Binary file with Content-Disposition | `200`, `404` |

#### Upload endpoint details (`POST /api/v1/resume` and `POST /api/v1/jobs/{id}/resume`):

- Accept `multipart/form-data` with a `file` part.
- Supported file types: `application/pdf`, `text/plain`. Return `400` with error code `UNSUPPORTED_FILE_TYPE` for anything else.
- For PDF files: extract text using **Apache PDFBox** (`org.apache.pdfbox:pdfbox`). Add this dependency to `build.gradle`:
  ```groovy
  implementation 'org.apache.pdfbox:pdfbox:3.0.4'
  ```
- For plain text files: read the bytes as UTF-8 string.
- Store both `file_content` (original bytes) and `content_text` (extracted text).
- If `content_text` extraction results in empty/blank text, return `400` with error code `EMPTY_RESUME_TEXT`.
- For the job-specific endpoint: validate job exists first (throw `JobNotFoundException` if not).

**Controller method signature pattern:**

```java
@PostMapping("/resume")
@ResponseStatus(HttpStatus.CREATED)
public ResumeResponse uploadMasterResume(@RequestParam("file") MultipartFile file) {
    return resumeService.uploadMasterResume(file);
}

@PostMapping("/jobs/{id}/resume")
@ResponseStatus(HttpStatus.CREATED)
public ResumeResponse uploadTailoredResume(@PathVariable UUID id,
                                            @RequestParam("file") MultipartFile file) {
    return resumeService.uploadTailoredResume(id, file);
}
```

#### Download endpoint details:

- Set `Content-Type` to the original file's content type.
- Set `Content-Disposition: attachment; filename="<original_file_name>"`.
- Return the raw `fileContent` bytes.
- Return `404` if no resume exists.

**Controller method signature pattern:**

```java
@GetMapping("/resume/download")

    // return ResponseEntity with headers and body
}
```

### Service Layer

#### `ResumeService.java`

Create in `src/main/java/com/jobtracker/service/ResumeService.java`.

```java
@Service
public class ResumeService {
    // Dependencies: ResumeRepository, JobRepository
    // Constructor injection (same pattern as JobService)
}
```

**Methods:**

1. `uploadMasterResume(MultipartFile file): ResumeResponse`
   - `@Transactional`
   - Validate file type (PDF or plain text only).
   - Extract text: PDF → PDFBox, text → read bytes as UTF-8.
   - Validate extracted text is not blank.
   - Create `Resume` entity with `jobId = null`.
   - Save and return `ResumeResponse`.

2. `uploadTailoredResume(UUID jobId, MultipartFile file): ResumeResponse`
   - `@Transactional`
   - Verify job exists (throw `JobNotFoundException` if not).
   - Same file processing as master.
   - Create `Resume` entity with `jobId` set.
   - Save and return `ResumeResponse`.

3. `getCurrentMasterResume(): ResumeResponse`
   - `@Transactional(readOnly = true)`
   - Call `resumeRepository.findFirstByJobIdIsNullOrderByCreatedAtDesc()`.
   - Return mapped `ResumeResponse` or `null`.

4. `getTailoredResume(UUID jobId): ResumeResponse`
   - `@Transactional(readOnly = true)`
   - Verify job exists.
   - Call `resumeRepository.findFirstByJobIdOrderByCreatedAtDesc(jobId)`.
   - Return mapped `ResumeResponse` or `null`.

5. `downloadMasterResume(): Resume`
   - `@Transactional(readOnly = true)`
   - Return full entity (including `fileContent`) or throw `ResumeNotFoundException`.

6. `downloadTailoredResume(UUID jobId): Resume`
   - `@Transactional(readOnly = true)`
   - Verify job exists.
   - Return full entity or throw `ResumeNotFoundException`.

7. `getResumeTextForScoring(UUID jobId): String`
   - `@Transactional(readOnly = true)`
   - This is the method `ResumeScorerAgent` will call.
   - Priority: tailored resume for this job → latest master → hardcoded fallback.
   - Returns the `contentText` string.

**Helper method for text extraction:**

```java
private String extractText(MultipartFile file) {
    String contentType = file.getContentType();
    if ("application/pdf".equals(contentType)) {
        try (PDDocument document = Loader.loadPDF(file.getBytes())) {
            return new PDFTextStripper().getText(document);
        } catch (IOException e) {
            throw new AgentException("Failed to extract text from PDF", e);
        }
    } else if ("text/plain".equals(contentType)) {
        return new String(file.getBytes(), StandardCharsets.UTF_8);
    } else {
        // This shouldn't happen if validation is done first, but safety net
        throw new IllegalArgumentException("Unsupported file type: " + contentType);
    }
}
```

**PDFBox imports:**
```java
import org.apache.pdfbox.Loader;
import org.apache.pdfbox.pdmodel.PDDocument;
import org.apache.pdfbox.text.PDFTextStripper;
```

### Modify `ResumeScorerAgent.java`

**Current state:** Has a hardcoded `RESUME_SUMMARY` constant and uses it directly in `buildUserPrompt()`.

**Required changes:**

1. Add `ResumeService` as a constructor dependency.
2. Change `score()` method signature to accept `jobId`:
   ```java
   public AgentInvocation<ScoreResult> score(UUID jobId, JdParseResult parsedJd)
   ```
3. In `buildUserPrompt()`, call `resumeService.getResumeTextForScoring(jobId)` instead of using `RESUME_SUMMARY`.
4. Keep `RESUME_SUMMARY` as the hardcoded fallback (used when no resume is uploaded).

**Updated `buildUserPrompt`:**
```java
private String buildUserPrompt(UUID jobId, JdParseResult parsedJd) {
    try {
        String jdJson = objectMapper.writeValueAsString(parsedJd);
        String resumeText = resumeService.getResumeTextForScoring(jobId);
        return "Parsed JD:\n" + jdJson + "\n\nCandidate resume:\n" + resumeText;
    } catch (JsonProcessingException ex) {
        throw new AgentException("Failed to serialize parsed JD for scoring prompt", ex);
    }
}
```

**Change to:**

## Feature 2: jdUrl Field

### Migration: `V7__add_jd_url_to_jobs.sql`

```sql
ALTER TABLE jobs
    ADD COLUMN jd_url VARCHAR(2048);
```

Use `VARCHAR(2048)` since URLs can be long.

### Model Changes

In `Job.java`, add:

```java
@Column(name = "jd_url")
private String jdUrl;

// Add getter and setter:
public String getJdUrl() { return jdUrl; }
public void setJdUrl(String jdUrl) { this.jdUrl = jdUrl; }
```

Place the field after `jdText` for logical grouping.

### DTO Changes

**`CreateJobRequest.java`** — add `jdUrl` field:
```java
public record CreateJobRequest(
        @NotBlank String company,
        @NotBlank String role,
        String jdText,

```

**`UpdateJobRequest.java`** — add `jdUrl` field (patchable):
```java
public record UpdateJobRequest(
        JobStatus status,
        String notes,
        String jdUrl        // <-- NEW: null = no change, empty string = clear
) {}
```

**`JobResponse.java`** — add `jdUrl` field:
```java
public record JobResponse(
        UUID id,
        String company,
        String role,
        JobStatus status,
        LocalDateTime appliedAt,
        String notes,
        String jdUrl,       // <-- NEW: between notes and fitScore
        Integer fitScore,
        LocalDateTime createdAt
) {}
```

### Mapper Changes

In `JobMapper.java`, add to the `toEntity` method:
```java
@Mapping(target = "jdUrl", source = "jdUrl")
```

This should be picked up automatically by MapStruct since field names match. However, since `unmappedTargetPolicy = ReportingPolicy.ERROR`, make sure the new field is accounted for. If MapStruct auto-maps it (same name in source and target), no explicit `@Mapping` is needed. If it fails compilation, add the explicit mapping.

### Service Changes

In `JobService.updateJob()`, add handling for `jdUrl` in the PATCH logic:

```java
if (request.jdUrl() != null) {
    job.setJdUrl(request.jdUrl().isEmpty() ? null : request.jdUrl());
}
```

Same pattern as `notes` — empty string clears the value.

### Postman Collection

Add `jdUrl` to relevant requests in `postman/job-tracker.postman_collection.json`:
- "Create Job - With JD Text" should include `"jdUrl": "https://example.com/careers/staff-engineer"`
