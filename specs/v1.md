# V1 Spec — Job Application Tracker

## Scope
Single endpoint to insert a job application record. Status defaults to UNDETERMINED — the record represents a job being evaluated, not necessarily one already applied to.

---

## Database Schema

### Table: `jobs`

| Column     | Type         | Constraints                                                                                                                                     |
|------------|--------------|-------------------------------------------------------------------------------------------------------------------------------------------------|
| id         | UUID         | PRIMARY KEY, default gen_random_uuid()                                                                                                          |
| company    | VARCHAR(255) | NOT NULL                                                                                                                                        |
| role       | VARCHAR(255) | NOT NULL                                                                                                                                        |
| jd_text    | TEXT         | nullable                                                                                                                                        |
| status     | VARCHAR(50)  | NOT NULL, default 'UNDETERMINED', CHECK (status IN ('UNDETERMINED','NOT_A_FIT','APPLIED','SCREENING','INTERVIEWING','OFFER_RECEIVED','OFFER_ACCEPTED','OFFER_DECLINED','REJECTED','WITHDRAWN','GHOSTED')) |
| applied_at | TIMESTAMP    | nullable                                                                                                                                        |
| created_at | TIMESTAMP    | NOT NULL, default now()                                                                                                                         |
| updated_at | TIMESTAMP    | NOT NULL, default now()                                                                                                                         |

---

## Job Status Lifecycle

Represented as a Java enum `JobStatus` mapped to VARCHAR in the DB via `@Enumerated(EnumType.STRING)`.

| Status | Meaning |
|---|---|
| `UNDETERMINED` | Just added, not reviewed yet — **default** |
| `NOT_A_FIT` | Match score too low, decided not to apply |
| `APPLIED` | Submitted the application |
| `SCREENING` | Recruiter screen scheduled or done |
| `INTERVIEWING` | In the interview process |
| `OFFER_RECEIVED` | Got an offer |
| `OFFER_ACCEPTED` | Accepted the offer |
| `OFFER_DECLINED` | Declined the offer |
| `REJECTED` | Rejected at any stage |
| `WITHDRAWN` | You pulled out of the process |
| `GHOSTED` | No response after a reasonable time |

## API

### POST /api/v1/jobs

Insert a new job record.

#### Request Body
```json
{
  "company": "Acme Corp",
  "role": "Senior Software Engineer",
  "jdText": "Optional job description text",
  "status": "UNDETERMINED",
  "appliedAt": "2026-04-06T10:00:00"
}
```

| Field     | Type          | Required | Notes                                                       |
|-----------|---------------|----------|-------------------------------------------------------------|
| company   | String        | Yes      |                                                             |
| role      | String        | Yes      |                                                             |
| jdText    | String        | No       | Maps to `jd_text` in DB                                     |
| status    | String        | No       | Must be a valid JobStatus value. Defaults to UNDETERMINED if omitted |
| appliedAt | LocalDateTime | No       | ISO 8601. Only meaningful when status is APPLIED or beyond  |

#### Response — 201 Created
```json
{
  "id": "a0eebc99-9c0b-4ef8-bb6d-6bb9bd380a11",
  "company": "Acme Corp",
  "role": "Senior Software Engineer",
  "status": "UNDETERMINED",
  "appliedAt": null,
  "createdAt": "2026-04-06T10:00:01"
}
```

| Field     | Type          | Notes           |
|-----------|---------------|-----------------|
| id        | UUID          | Auto-generated  |
| company   | String        |                 |
| role      | String        |                 |
| status    | String        | Enum name as string |
| appliedAt | LocalDateTime | Nullable        |
| createdAt | LocalDateTime |                 |

#### Error Responses

| Scenario                    | HTTP Status | Notes                               |
|-----------------------------|-------------|-------------------------------------|
| Missing `company` or `role` | 400         | Bean validation failure             |
| Invalid `status` value      | 400         | Must be a valid JobStatus enum value |
| Malformed JSON              | 400         |                                     |

---

## Business Rules
- `status` defaults to `UNDETERMINED` if not provided by the client
- `status` must be a valid `JobStatus` enum value — reject unknown values with 400
- `appliedAt` is optional and nullable — only meaningful when status is APPLIED or beyond
- `id`, `createdAt`, `updatedAt` are never accepted from the client — always server-generated
- `updatedAt` is not included in the V1 response
